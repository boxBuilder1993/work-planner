# WorkPlanner Active Data Access Layer

## Overview

The active data access layer handles all local reads and writes during normal app usage. It sits between the UI (ViewModels) and the local database, providing a clean, swappable abstraction.

```
┌─────────────────────────────────────────────────────┐
│                    ViewModels                        │
│              (observe + mutate)                      │
├─────────────────────────────────────────────────────┤
│                  TaskRepository                      │
│         (domain mapping, hierarchy helpers)           │
├──────────────┬──────────────────────────────────────┤
│   TaskDao    │   CommentDao                          │
│ (interface)  │   (interface)                         │
│   plain      │   plain                               │
│   Kotlin     │   Kotlin                              │
├──────────────┼──────────────────────────────────────┤
│ RoomTaskDao  │  RoomCommentDao                       │
│ (Room impl)  │  (Room impl)                          │
├──────────────┴──────────────────────────────────────┤
│              Room / SQLite                            │
└─────────────────────────────────────────────────────┘
```

- **TaskDao / CommentDao** — plain Kotlin interfaces. No annotations, no Room imports. Define what operations are available.
- **RoomTaskDao / RoomCommentDao** — extend the DAO interfaces, add Room annotations on every override. Generated by Room at compile time via KSP.
- **TaskRepository** — wraps DAOs, maps entities to domain models, provides hierarchy helpers (breadcrumbs). Injected with DAO interfaces — doesn't know it's Room underneath.
- **Swapping** — create a new `TaskDao` implementation, change the Hilt binding. Repository and ViewModels unchanged.

## Entities

Entity classes are shared between the active layer (Room) and backup layer (KVStore). They are annotated with both `@Entity` (Room) and `@Serializable` (Kotlin Serialization).

```kotlin
@Serializable
@Entity(
    tableName = "tasks",
    foreignKeys = [
        ForeignKey(
            entity = TaskEntity::class,
            parentColumns = ["id"],
            childColumns = ["parentId"],
            onDelete = ForeignKey.CASCADE
        )
    ],
    indices = [Index(value = ["parentId"])]
)
data class TaskEntity(
    @PrimaryKey
    val id: String = UUID.randomUUID().toString(),
    val parentId: String? = null,
    val title: String,
    val description: String = "",
    val status: String = "PENDING",       // "PENDING" or "CLOSED"
    val priority: Int = 3,                // 1 (highest) to 5 (lowest)
    val dueDate: Long? = null,            // epoch millis, null = no due date
    val createdAt: Long = System.currentTimeMillis(),
    val updatedAt: Long = System.currentTimeMillis()
)

@Serializable
@Entity(
    tableName = "comments",
    foreignKeys = [
        ForeignKey(
            entity = TaskEntity::class,
            parentColumns = ["id"],
            childColumns = ["taskId"],
            onDelete = ForeignKey.CASCADE
        )
    ],
    indices = [Index(value = ["taskId"])]
)
data class CommentEntity(
    @PrimaryKey
    val id: String = UUID.randomUUID().toString(),
    val taskId: String,
    val text: String,
    val createdAt: Long = System.currentTimeMillis(),
    val updatedAt: Long = System.currentTimeMillis()
)
```

## DAO Interfaces (Annotation-Free)

Plain Kotlin interfaces. No Room dependency. Define the contract for data operations.

### TaskDao

```kotlin
interface TaskDao {

    // ── Reactive queries (return Flow for UI observation) ─────

    // Get root-level tasks (themes) — parentId is null
    fun getRootTasks(): Flow<List<TaskEntity>>

    // Get PENDING root-level tasks (for Themes tab)
    fun getPendingRootTasks(): Flow<List<TaskEntity>>

    // Get direct children of a task
    fun getChildTasks(parentId: String): Flow<List<TaskEntity>>

    // Get a single task by ID (reactive)
    fun getTaskById(id: String): Flow<TaskEntity?>

    // Count direct children
    fun getChildCount(parentId: String): Flow<Int>

    // Get PENDING leaf tasks (no children) — for Actionable tab
    fun getLeafTasks(): Flow<List<TaskEntity>>

    // Search tasks by title or description — for Search tab
    fun searchTasks(query: String): Flow<List<TaskEntity>>

    // ── One-shot queries (suspend, for non-UI operations) ────

    // Get a single task by ID (one-shot, for breadcrumb walking)
    suspend fun getTaskByIdOnce(id: String): TaskEntity?

    // Get all tasks (for backup export)
    suspend fun getAllTasks(): List<TaskEntity>

    // ── Mutations ────────────────────────────────────────────

    suspend fun insertTask(task: TaskEntity)

    suspend fun insertTasks(tasks: List<TaskEntity>)

    suspend fun updateTask(task: TaskEntity)

    suspend fun deleteTaskById(id: String)

    // Clear all tasks (for restore from backup)
    suspend fun deleteAllTasks()
}
```

## Room Implementations

Extend the DAO interfaces, add Room annotations on every override. Room generates the implementation at compile time via KSP.

### RoomTaskDao

```kotlin
@Dao
interface RoomTaskDao : TaskDao {

    // ── Reactive queries ─────────────────────────────────────

    @Query("SELECT * FROM tasks WHERE parentId IS NULL ORDER BY createdAt ASC")
    override fun getRootTasks(): Flow<List<TaskEntity>>

    @Query("SELECT * FROM tasks WHERE parentId IS NULL AND status = 'PENDING' ORDER BY createdAt ASC")
    override fun getPendingRootTasks(): Flow<List<TaskEntity>>

    @Query("SELECT * FROM tasks WHERE parentId = :parentId ORDER BY createdAt ASC")
    override fun getChildTasks(parentId: String): Flow<List<TaskEntity>>

    @Query("SELECT * FROM tasks WHERE id = :id")
    override fun getTaskById(id: String): Flow<TaskEntity?>

    @Query("SELECT COUNT(*) FROM tasks WHERE parentId = :parentId")
    override fun getChildCount(parentId: String): Flow<Int>

    @Query("SELECT * FROM tasks WHERE status = 'PENDING' AND id NOT IN (SELECT DISTINCT parentId FROM tasks WHERE parentId IS NOT NULL) ORDER BY priority ASC, CASE WHEN dueDate IS NULL THEN 1 ELSE 0 END, dueDate DESC")
    override fun getLeafTasks(): Flow<List<TaskEntity>>

    @Query("SELECT * FROM tasks WHERE title LIKE '%' || :query || '%' OR description LIKE '%' || :query || '%' ORDER BY createdAt ASC")
    override fun searchTasks(query: String): Flow<List<TaskEntity>>

    // ── One-shot queries ─────────────────────────────────────

    @Query("SELECT * FROM tasks WHERE id = :id")
    override suspend fun getTaskByIdOnce(id: String): TaskEntity?

    @Query("SELECT * FROM tasks")
    override suspend fun getAllTasks(): List<TaskEntity>

    // ── Mutations ────────────────────────────────────────────

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    override suspend fun insertTask(task: TaskEntity)

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    override suspend fun insertTasks(tasks: List<TaskEntity>)

    @Update
    override suspend fun updateTask(task: TaskEntity)

    @Query("DELETE FROM tasks WHERE id = :id")
    override suspend fun deleteTaskById(id: String)

    @Query("DELETE FROM tasks")
    override suspend fun deleteAllTasks()
}
```

### Entity ↔ Domain Mapping

```kotlin
fun TaskEntity.toDomain() = Task(
    id, parentId, title, description,
    TaskStatus.valueOf(status), priority, dueDate, createdAt, updatedAt
)

fun Task.toEntity() = TaskEntity(
    id, parentId, title, description,
    status.name, priority, dueDate, createdAt, updatedAt
)

fun CommentEntity.toDomain() = Comment(id, taskId, text, createdAt, updatedAt)

fun Comment.toEntity() = CommentEntity(id, taskId, text, createdAt, updatedAt)
```

## TaskRepository

The single access point for all task-related operations. Wraps DAOs, maps to domain models, provides hierarchy helpers. Injected with DAO interfaces.

```kotlin
class TaskRepository(
    private val taskDao: TaskDao,
    private val commentDao: CommentDao
) {

    // ── Task queries ─────────────────────────────────────────

    fun getRootTasks(): Flow<List<Task>> =
        taskDao.getRootTasks().map { list -> list.map { it.toDomain() } }

    fun getPendingRootTasks(): Flow<List<Task>> =
        taskDao.getPendingRootTasks().map { list -> list.map { it.toDomain() } }

    fun getChildTasks(parentId: String): Flow<List<Task>> =
        taskDao.getChildTasks(parentId).map { list -> list.map { it.toDomain() } }

    fun getTaskById(taskId: String): Flow<Task?> =
        taskDao.getTaskById(taskId).map { it?.toDomain() }

    fun getChildCount(taskId: String): Flow<Int> =
        taskDao.getChildCount(taskId)

    fun getLeafTasks(): Flow<List<Task>> =
        taskDao.getLeafTasks().map { list -> list.map { it.toDomain() } }

    fun searchTasks(query: String): Flow<List<Task>> =
        taskDao.searchTasks(query).map { list -> list.map { it.toDomain() } }

    // ── Task mutations ───────────────────────────────────────

    suspend fun createTask(
        title: String,
        description: String,
        parentId: String?,
        priority: Int = 3,
        dueDate: Long? = null
    ): Task {
        val entity = TaskEntity(
            title = title,
            description = description,
            parentId = parentId,
            priority = priority,
            dueDate = dueDate
        )
        taskDao.insertTask(entity)
        return entity.toDomain()
    }

    suspend fun updateTask(task: Task) {
        taskDao.updateTask(task.toEntity().copy(updatedAt = System.currentTimeMillis()))
    }

    suspend fun closeTask(taskId: String) {
        val task = taskDao.getTaskByIdOnce(taskId) ?: return
        taskDao.updateTask(task.copy(
            status = "CLOSED",
            updatedAt = System.currentTimeMillis()
        ))
    }

    suspend fun reopenTask(taskId: String) {
        val task = taskDao.getTaskByIdOnce(taskId) ?: return
        taskDao.updateTask(task.copy(
            status = "PENDING",
            updatedAt = System.currentTimeMillis()
        ))
    }

    suspend fun reparentTask(taskId: String, newParentId: String?) {
        val task = taskDao.getTaskByIdOnce(taskId) ?: return
        taskDao.updateTask(task.copy(
            parentId = newParentId,
            updatedAt = System.currentTimeMillis()
        ))
    }

    suspend fun deleteTask(taskId: String) {
        taskDao.deleteTaskById(taskId)  // CASCADE handles children + comments
    }

    // ── Comment queries ──────────────────────────────────────

    fun getCommentsForTask(taskId: String): Flow<List<Comment>> =
        commentDao.getCommentsForTask(taskId).map { list -> list.map { it.toDomain() } }

    // ── Comment mutations ────────────────────────────────────

    suspend fun addComment(taskId: String, text: String): Comment {
        val entity = CommentEntity(taskId = taskId, text = text)
        commentDao.insertComment(entity)
        return entity.toDomain()
    }

    suspend fun deleteComment(comment: Comment) {
        commentDao.deleteComment(comment.toEntity())
    }

    // ── Hierarchy helpers ────────────────────────────────────

    suspend fun getBreadcrumbs(taskId: String): List<Task> {
        val crumbs = mutableListOf<Task>()
        var currentId: String? = taskId
        while (currentId != null) {
            val task = taskDao.getTaskByIdOnce(currentId) ?: break
            crumbs.add(0, task.toDomain())  // prepend
            currentId = task.parentId
        }
        return crumbs
    }
}
```

## Dependency Injection

```kotlin
@Module
@InstallIn(SingletonComponent::class)
object AppModule {

    @Provides
    @Singleton
    fun provideDatabase(@ApplicationContext context: Context): WorkPlannerDatabase {
        return Room.databaseBuilder(
            context,
            WorkPlannerDatabase::class.java,
            "workplanner.db"
        ).build()
    }

    // Provide DAO interfaces, backed by Room implementations
    @Provides
    fun provideTaskDao(db: WorkPlannerDatabase): TaskDao = db.roomTaskDao()

    @Provides
    fun provideCommentDao(db: WorkPlannerDatabase): CommentDao = db.roomCommentDao()

    @Provides
    @Singleton
    fun provideTaskRepository(taskDao: TaskDao, commentDao: CommentDao): TaskRepository {
        return TaskRepository(taskDao, commentDao)
    }
}
```

### Swapping the Backend

To replace Room with another local storage:

1. Create new `TaskDao` / `CommentDao` implementations (e.g., `ApiTaskDao`, `RealmTaskDao`).
2. Change the Hilt bindings in `AppModule`.
3. `TaskRepository`, ViewModels, and UI remain unchanged.

```kotlin
// Today — Room
@Provides fun provideTaskDao(db: WorkPlannerDatabase): TaskDao = db.roomTaskDao()

// Tomorrow — Remote API
@Provides fun provideTaskDao(apiClient: ApiClient): TaskDao = ApiTaskDao(apiClient)
```

## How Active and Backup Layers Connect

The `BackupProcessor` (defined in [BACKUP_DATA_ACCESS.md](./BACKUP_DATA_ACCESS.md)) uses the same DAO interfaces to read from and write to the local store:

```
                    ┌─────────────┐
                    │ TaskRepository │ ← ViewModels use this
                    └──────┬──────┘
                           │
                    ┌──────┴──────┐
                    │  TaskDao     │ ← plain interface
                    │  CommentDao  │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
        ┌─────┴─────┐     │     ┌──────┴──────┐
        │ RoomTaskDao│     │     │BackupProcessor│
        │ (active    │     │     │(reads DAOs,   │
        │  storage)  │     │     │ writes KVStore)│
        └────────────┘     │     └──────┬──────┘
                           │            │
                           │      ┌─────┴─────┐
                           │      │  KVStore   │
                           │      │(GDriveKV)  │
                           │      └────────────┘
                           │
                    ┌──────┴──────┐
                    │   SQLite     │
                    └─────────────┘
```

The BackupProcessor is injected with the same `TaskDao` and `CommentDao` interfaces. It calls `getAllTasks()` / `getAllComments()` for backup, and `deleteAllTasks()` / `insertTasks()` for restore.
